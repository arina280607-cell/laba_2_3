### Реализованные требования

#### 1. Протокол итерации (Итераторы)
- Класс **TaskQueue** реализует метод `__iter__`, что позволяет использовать объект очереди в циклах `for` и стандартных функциях Python (`list()`, `max()`, `sorted()`).
- Коллекция поддерживает **повторную итерацию**, не теряя данные при обходе.

#### 2. Магические методы
- Реализован `__len__` для получения количества задач в очереди через `len(queue)`.
- Реализован `__repr__` для информативного отображения состояния объекта очереди.

#### 3. Ленивые вычисления (Генераторы)
- **filter_by_status(status)** — возвращает генератор задач с определенным статусом.
- **filter_by_priority(min_priority)** — ленивая фильтрация задач по уровню важности.
- **ready_tasks()** — генератор задач, готовых к выполнению (`is_ready`).
- Использование ключевого слова `yield` обеспечивает экономию памяти: фильтрация происходит "на лету" во время итерации.

#### 4. Интерактивный CLI
- Поддержка объединения задач из разных источников (JSON + Ручной ввод) в одной очереди.
- Меню управления с возможностью многократной фильтрации и просмотра данных.

### Основной функционал
1. **Модель Task** (src/contracts/tasks.py): Валидация данных через дескрипторы.
2. **TaskQueue** (src/inbox/queue.py): Умная коллекция с поддержкой генераторов.
3. **InboxApp** (src/inbox/core.py): Логика загрузки данных из источников.
4. **CLI** (src/cli.py): Интерфейс взаимодействия с пользователем.
